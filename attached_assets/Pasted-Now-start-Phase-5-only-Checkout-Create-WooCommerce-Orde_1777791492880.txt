Now start Phase 5 only: Checkout + Create WooCommerce Order.

The storefront already has:
- Next.js foundation
- Premium UI
- WooCommerce products and categories integration
- Product details and variations support
- Local cart system with Zustand and localStorage

Do not rebuild the whole project.
Do not change the overall design direction.
Do not implement full payment gateway integration manually.
Do not implement customer account/authentication yet.
Do not implement the custom WordPress plugin yet.

Goal:
Build a secure checkout flow that takes the current cart, collects customer billing/shipping details, creates a WooCommerce order server-side, and redirects the customer to the WooCommerce payment URL.

Important:
WooCommerce must remain the source of truth for prices, totals, taxes, shipping, coupons, and order status.

Critical Security Rule:
Never trust product prices, totals, discounts, or shipping values from the browser/localStorage.
The browser should only send:
- product_id
- variation_id if available
- quantity
- selected attributes if needed for validation
- customer billing/shipping fields
- coupon code if available
- customer note if available

The server must validate and create the order using WooCommerce REST API.

3. Checkout API Route

Create secure server route:

app/api/checkout/create-order/route.ts

This API route should:
- Accept POST only.
- Read cart items and customer data from request body.
- Validate request body using Zod.
- Reject empty cart.
- Reject invalid product IDs.
- Reject invalid quantities.
- Never trust price values from client.
- Create WooCommerce order server-side using API credentials from environment variables.
- Return payment_url and order_id.
- Handle errors cleanly.

4. WooCommerce Order Creation

Use WooCommerce REST API:

POST /wp-json/wc/v3/orders

Order payload should include:
- payment_method
- payment_method_title
- set_paid: false
- billing data
- shipping data
- line_items:
  - product_id
  - variation_id if exists
  - quantity
- coupon_lines if coupon exists
- customer_note if exists
- status: pending

Important:
Do not send prices from frontend.
Let WooCommerce calculate item prices from product IDs and variation IDs.

5. Payment Method Handling

For this phase:

- The goal is to create the order and get WooCommerce payment_url.

If payment_url exists:
- Redirect customer to payment_url.

If payment_url does not exist:
- If payment method is Cash on Delivery, redirect to local order success/pending page.
- Otherwise show a clear error.
