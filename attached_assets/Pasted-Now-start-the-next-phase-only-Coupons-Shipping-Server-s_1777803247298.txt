Required Work:

1. Server-side Cart/Totals Calculation

Create a secure Next.js API route:

POST /api/cart/calculate

Input from browser:
- line_items:
  - product_id
  - variation_id optional
  - quantity
- billing/shipping address if available
- coupon_code optional

Do not accept:
- product prices
- subtotal
- discount total
- shipping total
- tax total
- final total

The API route should:
- Validate input using Zod.
- Fetch product/variation data from WooCommerce server-side.
- Verify products exist.
- Verify variations belong to products.
- Verify stock status if possible.
- Validate quantities.
- Calculate subtotal server-side.
- Validate coupon code with WooCommerce.
- Calculate discount.
- Fetch or calculate available shipping methods if possible.
- Return safe calculated totals.

If exact WooCommerce cart calculation is hard via REST API:
- Implement a safe approximation using WooCommerce product prices server-side.
- Validate coupon using WooCommerce coupon endpoint.
- Let final exact total be calculated by WooCommerce during order creation.
- Clearly keep order creation as final source of truth.

2. Coupon Support

Add coupon support in Cart and Checkout.

Server requirements:
- Coupon validation must happen server-side.
- Use WooCommerce coupon API:
  GET /wp-json/wc/v3/coupons?code={couponCode}
- Check:
  - coupon exists
  - coupon is not expired
  - usage limit if available
  - minimum amount if available
  - maximum amount if available
  - product/category restrictions if practical
  - excluded products/categories if practical
- Return safe coupon data only:
  - code
  - discount_type
  - amount
  - description if safe
  - calculated_discount

Important:
Do not trust coupon discount calculated in the browser.

Server requirements:
- Add API route:
  POST /api/shipping/methods

Input:
- cart line items
- shipping address

The API route should:
- Validate request.
- Determine available shipping methods based on WooCommerce settings if possible.
- Return safe shipping methods.

Important:
WooCommerce REST API does not always provide direct live cart shipping calculation easily.
If live shipping rate calculation is not available through current setup, implement a practical first version:
- Read WooCommerce shipping zones/methods server-side.
- Match country/state/postcode if possible.
- Return enabled flat_rate/free_shipping/local_pickup methods.
- Let WooCommerce order creation remain the final authority.
- Make the code extendable for advanced shipping later.
