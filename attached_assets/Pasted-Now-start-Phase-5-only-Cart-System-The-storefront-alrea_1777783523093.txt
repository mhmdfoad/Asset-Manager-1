Now start Phase 5 only: Cart System.

The storefront already has:
- Next.js foundation
- Premium UI
- WooCommerce products and categories integration
- Product details page
- WooCommerce variations support

Do not rebuild the whole project.
Do not change the overall design direction.
Do not implement checkout/payment yet.
Do not implement authentication yet.
Do not implement the custom WordPress plugin yet.

Goal:
Build a clean, production-ready cart system for the Next.js Headless WooCommerce storefront.

Important:
In this phase, we only implement cart functionality.
Checkout and order creation will be implemented in the next phase.

Cart Strategy:
Use a Next.js local cart state first, stored with Zustand and persisted to localStorage.
The cart should store only safe product references and quantities, not trusted prices.

Important Security Rule:
Never trust prices from localStorage or the browser during checkout.
For now, prices can be displayed from the product/variation data for UI purposes.
In the next checkout phase, prices must be recalculated server-side from WooCommerce.

Required Work:

1. Cart Store

Create a Zustand cart store in:

store/cart-store.ts

The cart store should support:
- Add item
- Remove item
- Update quantity
- Increase quantity
- Decrease quantity
- Clear cart
- Get total item count
- Get subtotal for display
- Check if item already exists
- Persist cart to localStorage

Cart item structure should include:
- product_id
- variation_id optional
- slug
- name
- image
- selected_attributes optional
- quantity
- price_for_display only
- regular_price_for_display optional
- sale_price_for_display optional
- stock_status optional
- max_quantity optional if available

Important:
price_for_display is only for UI display.
Do not treat it as trusted for checkout.

2. Add to Cart from Product Page

Update the product details page:

For simple products:
- Add to Cart button should add product_id and quantity.
- Quantity selector should be available.
- If product is out of stock, disable the button.

For variable products:
- Add to Cart button should only work after a valid variation is selected.
- Store selected variation_id.
- Store selected attributes.
- If variation is out of stock, disable the button.
- Show a clear message if user tries to add without selecting all required attributes.
- Quantity selector should respect stock if available.

3. Cart Drawer

Create a modern cart drawer component:

components/cart/CartDrawer.tsx

Requirements:
- Opens when clicking the cart icon in the header.
- Shows cart items.
- Shows product image, name, selected attributes, quantity controls, price, remove button.
- Shows subtotal for display.
- Shows buttons:
  - View Cart
  - Checkout
- Checkout button can link to /checkout placeholder for now.
- Empty cart state.
- Smooth open/close animation if the project already uses animations.
- RTL/LTR compatible.
- Mobile-friendly.
