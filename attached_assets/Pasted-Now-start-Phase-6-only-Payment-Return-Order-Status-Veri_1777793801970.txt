Now start Phase 6 only: Payment Return + Order Status Verification.

Current situation:
The checkout flow is working correctly:
- The customer fills the Next.js checkout form.
- Next.js creates a WooCommerce order.
- The customer is redirected to the WooCommerce payment URL:
  /checkout/order-pay/{order_id}/?pay_for_order=true&key={order_key}
- The customer completes payment successfully.
- WooCommerce currently redirects the customer to the default WooCommerce Thank You page.

Problem:
After payment, I want the customer to return back to the Next.js storefront, not stay on the WooCommerce Thank You page.

Goal:
After payment success, failed payment, cancelled payment, or pending payment, WooCommerce should redirect the customer back to the correct Next.js page.

Required final flow:
1. Customer fills checkout in Next.js.
2. Next.js creates WooCommerce order.
3. Customer is redirected to WooCommerce order-pay page.
4. Customer pays using WooCommerce payment gateway.
5. WooCommerce processes the payment.
6. WooCommerce redirects customer back to Next.js:
   - Success:
     /order/success?order_id={order_id}&key={order_key}
   - Failed:
     /order/failed?order_id={order_id}&key={order_key}
   - Pending:
     /order/pending?order_id={order_id}&key={order_key}
7. Next.js verifies the order status server-side by calling WooCommerce API.
8. Next.js displays the correct result page.
9. If the order is paid/processing/completed, clear the local cart.
10. If payment failed or is pending, do not clear the cart too early.

Important:
Do not trust query parameters alone.
The Next.js success page must verify the real order status from WooCommerce server-side.

6. Failed and pending payment handling

If WooCommerce/gateway marks order failed:
- Redirect to:
  {frontend_url}/order/failed?order_id={order_id}&key={order_key}

If order remains pending or on-hold:
- Redirect to:
  {frontend_url}/order/pending?order_id={order_id}&key={order_key}

If exact gateway failed redirect is hard to detect:
- Redirect all returns to:
  /order/verify?order_id={order_id}&key={order_key}
Then Next.js checks the real status and internally displays success, failed, or pending.

Recommended simpler approach:
Use one universal page:
  /order/verify?order_id={order_id}&key={order_key}

This page verifies status and then shows or redirects to:
- /order/success
- /order/failed
- /order/pending

7. Recommended implementation

Use this robust approach:
- WooCommerce redirects every completed payment return to:
  /order/verify?order_id={order_id}&key={order_key}
- Next.js verifies the actual WooCommerce order status.
- Next.js decides what to show:
  - processing/completed = success
  - failed/cancelled = failed
  - pending/on-hold = pending

This is better because it does not rely on WooCommerce guessing the final page.
