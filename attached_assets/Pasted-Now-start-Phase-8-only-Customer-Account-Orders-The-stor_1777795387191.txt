Goal:
Build a secure customer account system in the Next.js storefront connected to WordPress/WooCommerce.

The customer should be able to:
- Register
- Login
- Logout
- View account dashboard
- View order history
- View order details
- View and update billing/shipping addresses
- View and update basic profile information

Important:
WordPress/WooCommerce remains the source of truth for customers and orders.

Authentication Strategy:
Use secure server-side authentication.

Preferred approach:
- Customer logs in from Next.js.
- Next.js API route sends credentials to WordPress JWT endpoint.
- WordPress returns JWT token.
- Next.js stores the JWT in an HTTP-only secure cookie.
- The browser must not store JWT in localStorage.
- Authenticated server-side routes use this token to fetch WooCommerce customer data.

Required WordPress dependency:
Use JWT Authentication for WP REST API or an equivalent secure JWT plugin.

Required Environment Variables:
WORDPRESS_URL=
WOOCOMMERCE_STORE_URL=
WOOCOMMERCE_CONSUMER_KEY=
WOOCOMMERCE_CONSUMER_SECRET=
JWT_AUTH_SECRET_KEY if needed

Required Work:

1. Auth API Routes

Create these Next.js API routes:

app/api/auth/login/route.ts
app/api/auth/register/route.ts
app/api/auth/logout/route.ts
app/api/auth/me/route.ts

Login route:
- Accept email/username and password.
- Validate input with Zod.
- Call WordPress JWT token endpoint server-side.
- Store JWT token in HTTP-only cookie.
- Return safe user data only.
- Handle invalid credentials.

Register route:
- Accept first name, last name, email, password, phone optional.
- Validate input with Zod.
- Create WooCommerce customer using REST API server-side.
- Optionally auto-login after registration.
- Handle duplicate email error.

Logout route:
- Clear auth cookie.
- Return success.

Me route:
- Read JWT from HTTP-only cookie.
- Verify token with WordPress if needed.
- Return safe logged-in customer data.

Security:
- Never expose JWT token to browser JavaScript.
- Never store JWT in localStorage.
- Use HTTP-only, secure, SameSite cookie.
- Validate all request bodies.
- Rate limit login/register if possible.
